# NAVER LABS 브랜드 규칙 (생성 시 강제)

근거: NAVER LABS CI Simple Guidelines(2020.06.16), Company Profile, naverlabs.com CSS 실측.

## 컬러

| 토큰 | 값 | 용도 |
|---|---|---|
| `--nl-blue` | `#006EFF` | 시그니처 포인트. 강조 텍스트, 수치, 타임라인 도트, 다이어그램 글로우 |
| `--nl-black` | `#0F0F0F` | 로고 블랙, 다이어그램 배경 |
| `--nl-gray` | `#B2B2B2` | 보조 로고 컬러 |

- **주의: 포인트 컬러는 파랑이다.** 네이버 본사의 그린을 강조색으로 쓰지 않는다.
- 그린 계열은 커버 그라디언트(`linear-gradient(135deg, #dcf76d 0%, #00e06e 45%, #00b6f0 100%)`)에서만 사용.
- 일반 슬라이드는 시맨틱 토큰(`--surface`, `--surface-alt`, `--ink`, `--ink-sub`, `--ink-faint`, `--line`, `--accent`)만 참조한다. 색상 하드코딩 금지.

## 타이포그래피

- 폰트 스택: `"NanumHuman", "Pretendard", "Malgun Gothic", sans-serif` (Apple SD Gothic Neo 금지)
- 자간: 본문 `-0.3px`, 헤드라인 `-1px ~ -1.5px` (naverlabs.com 실측)
- 헤드라인은 영문 우선(Company Profile 관례), 본문·설명은 한글
- **문구 톤: 개조식(명사형 종결) 원칙.** 발표 슬라이드는 "~합니다/~했습니다/~입니다" 같은 서술형 종결을 피하고, 명사·명사형으로 끝맺어 간결하게(예: "…설계·구현", "…주도", "…경험", "…해결"). 카드/불릿/리드 모두 동일.
- 웨이트: 900(히어로/숫자), 800(타이틀), 700(카드 제목/킥커), 400(본문)

## 레이아웃

- 캔버스 1280×720 고정, 슬라이드 패딩 72px 80px
- 여백을 아끼지 않는다. 슬라이드당 핵심 메시지 1개
- 슬라이드 유형별 배경: 화이트(콘텐츠) / `--surface-alt`(목차) / 블랙(다이어그램) / 그라디언트(커버)

## 로고

- `assets/logo.svg` 인라인 사용, `currentColor` 상속 (`.logo` 래퍼의 color로 제어)
- 최소 너비: 디지털 11px (CI 가이드), 슬라이드에선 220px 이상 권장
- 로고 변형·왜곡·재조합 금지 (CI 가이드 준수)
