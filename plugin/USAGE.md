# 사용 가이드

## 스킬 호출

자연어로 요청하면 Claude가 자동으로 스킬을 사용합니다:

> 네이버랩스 스타일로 발표자료 만들어줘
>
> (내용 첨부하며) 이 내용으로 네이버랩스 CI 슬라이드 만들어줘

명시적으로 호출하려면: `/naverlabs-slides:naverlabs-slides`

기존 덱 수정도 됩니다:

> 5번 슬라이드를 stat 형으로 바꿔줘
>
> 마지막에 감사 슬라이드 추가해줘

## 생성 흐름

1. **내용 수집** — 대화·첨부에서 발표 내용을 모읍니다. 부족하면 발표 시간·청중·핵심 메시지를 묻습니다.
2. **아키타입 매핑** — 내용을 슬라이드 유형에 매핑하고, 10장 초과 시 목차(toc)를 커버 다음 2번째에 넣습니다. **구성안을 먼저 보여주고 승인**을 받습니다.
3. **조립** — 단일 HTML로 출력합니다 (기본 파일명 `<주제>-deck.html`).
4. **검증** — 외부 URL 0건, 플레이스홀더 0건을 자동으로 확인합니다.

## 아키타입 (슬라이드 유형)

| 유형 | 쓰임 |
|---|---|
| `cover` | 표지 (제목 · 발표자) |
| `toc` | 목차 |
| `divider` | 챕터 전환 |
| `diagram` | 개념 관계 · 아키텍처 |
| `stat` | 수치 성과 (%, 배수, 개수) |
| `timeline` | 시간순 이력 |
| `content` | 일반 본문 (grid-2 / grid-3 / card) |
| `closing` | 마무리 · 감사 |

## 슬라이드 조작

- **우측 상단 컨트롤** — 모드 토글(스크롤 ↔ 발표) / 테마(라이트 ↔ 다크) / 페이지 인디케이터(클릭 → 번호 입력)
- **발표 모드** — 방향키·Space로 진행, 숫자 + Enter로 점프, Esc로 스크롤 복귀, 모바일 스와이프
- **URL 파라미터** — `#5`(5페이지로 이동), `?mode=present`(발표 모드), `?theme=dark`(다크 테마)

## PDF 내보내기

브라우저 인쇄(**⌘P / Ctrl+P**) → 슬라이드당 1페이지로 저장됩니다.
제출·인쇄용은 **라이트 테마**를 권장합니다 (기본값).

## 브랜드 규칙 (덱 품질 유지)

- 포인트 컬러는 `#006EFF` 블루. 그린 계열은 커버 그라디언트 전용
- 기본 테마는 **라이트** — 밝은 회의실 투사·PDF 제출에 유리
- 캔버스(1280×720) 오버플로 금지 — 카드 본문 3줄, 불릿 4개 이내
- 각 슬라이드에 `data-section="대분류"`를 주면 하단에 NAVER LABS 로고 + 대분류 라벨 푸터가 자동으로 붙습니다 (cover·toc·closing 제외)

## 오프라인 자립성

생성된 HTML은 이미지·폰트·스크립트를 모두 인라인으로 담습니다. 인터넷 없이도 열리고, USB·이메일로 파일 하나만 전달하면 됩니다. (단, `<a href>` 내비게이션 링크는 클릭 시에만 이동합니다.)
